B3- Normalisation
To satisfy N1 normalisation, we need to make sure that the values in table are atomic. To
achieve this, we have written User_contact table separately instead of adding it with users.

create table user_contact(
contact_id integer primary key autoincrement,
user_id integer,
contact_no text,
foreign key (user_id) references user(user_id)
);
Also, as organisation may have multiple purchases, to make it atomic, we have made the
following tablecreate table organisation_plan(
organisation_plan_id integer primary key autoincrement,
organisation_id integer,
plan_id integer,
starting_date text,
ending_date text,
foreign key (organisation_id) references organisation(organisation_id),
foreign key (plan_id) references plan(plan_id)
);
Since we have only 1 primary key for each table, and it satisfies NF1, we can say that each table
satisfies NF2.
Also, table is made in such a way that no non primary key is depended on other non-primary
key.Thus it satisfies NF3.
